Mozilla Servo Open Source Contributor

March 2018 - Present

After learning the fundamentals of Rust while working on some personal projects, I decided to get involved with Rust's amazing open source community by contributing to Mozilla's Servo project. This has been a great experience that has led me to learn more about Rust as a language and gain experience working on a prototype browser engine! The first feature I worked on was to update Servo's memory profiling to accurately measure the heap usage of the JavaScript Document Object Model for each loaded webpage. To achieve this, I updated the Rust to C++ Foreign Function Interface to allow use of the SpiderMonkey API to measure DOM objects. This improved the accuracy of memory reporting for DOM objects by 33% and my contribution was featured as a Noteable Addition in This Week in Servo. I am currently working on implementing a compiler plugin which parses WebIDL files to verify the inheritance of DOM objects.

Spotify User Data Graph

February 2018 - Present

I am very interested in graph databases and how graph structures can be used to capture relationships in data. In an effort to develop a graph that can lookup both nodes and edges in constant time, I implemented my own graph using a hybrid indexing approach. I ran some basic benchmarking tests on my structure and showed that, for a graph with 100 up to ~3000 nodes and edges between every node, the time required for each operation was approximately constant (the upper bound is due to memory restrictions on my laptop). I continue to explore this area and I am going to use the graph to encapsulate proximity of play relationships for Spotify songs. I have developed a basic clustering algorithm which groups songs based on the weightings of edges with other songs in the group. I am currently working on improving this algorithm and converting it to be multithreaded. I would also like to develop more features for the graph such as caching clusters. You can check out the project on Github.

DoppelGallery - Hack the North 2017 Winner!

September 2017

I worked as part of a team of four at Hack the North to develop DoppelGallery, a web app which matches your face to people in historical paintings and photographs. I worked on the selection of the facial recognition API and the backend, using CockroachDB to store image data for model training and the web app. You can check out the project on DevPost and GitHub.
